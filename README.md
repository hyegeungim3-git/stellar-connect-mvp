# 내 아이 설명서 (Stellar Connect · S:CON) — 1차 MVP

발달장애 아이를 **누구나 이해할 수 있도록** 돕는 서비스. 매일 기록을 쌓는 앱이 아니라,
부모를 대신해 우리 아이를 설명해 주는 **결과물(대상별 설명서)** 을 만들어 주는 것을 목표로 합니다.

> 기록을 남기는 서비스가 아니라, **아이를 이해하게 만드는** 서비스입니다.

- 라이브: https://hyegeungim3-git.github.io/stellar-connect-mvp/
- 체험 계정: `parent@example.com` / `1234` (관리자 `admin@ichild.kr` / `admin123`)

## 차별화

1. **아이를 이해하는 플랫폼** — 좋아하는 것/싫어하는 것, 의사소통 방법, 감각 특성,
   도전적 행동 발생 상황과 효과적 지원 방법을 8개 카테고리로 정리
2. **결과물 = 대상별 「내 아이 설명서」** — 같은 데이터를 **학교용 / 병원용 / 활동지원사용 /
   돌봄기관용**으로 자동 구성해 한 장으로 공유. 대상별 구성(포함 항목·순서)을 직접 편집하거나
   새 대상을 추가할 수 있습니다
3. **보호자 중심 UX** — 최소 입력 · 빠른 입력 칩 · 음성 입력 · 쉬운 공유(QR·링크·4자리 인증·안심번호)
4. **생애주기 미래 준비** — 영유아기(0~6세) → 학령기(7~12세) → 청소년기(13~18세) → 성인 준비기(19세~)

## 화면

**좌측 메뉴 8개** — 홈 · 아이 프로필 · 설명서 · 기록 · 복용 관리 · 갤러리 · 미래 준비 · 대상별 공유
(양육자 정보는 우측 상단 계정 드롭다운에서 진입)

**모바일 하단 탭 4개** — 홈 · 설명서 · 기록 · 갤러리 (+ 더보기 드로어)

주요 기능

- **설명서** — 8개 카테고리(할 수 있어요 · 도움이 필요해요 · 좋아해요 · 싫어해요 ·
  도전적 행동 및 대응 · 의사소통 방법 · 생활 루틴 · 안전 주의사항), 한 줄 소개, 보호자 한마디
- **한 장 요약** — 응급 정보·알레르기를 항상 최상단 고정, 인쇄(PDF 저장) 지원
- **대상별 공유** — 공유 기간(1일/1주/1개월/1년/계속) · 4자리 인증번호 · 안심번호(가상 0508) ·
  QR·키링 카드(86×54mm 인쇄)
- **기록** — 행동 · 치료 · 복용 · 변화 · 검사/평가 5종. 검색과 기간·컨디션·태그·정렬 필터
- **복용 관리** — 약물 9필드 등록, 시간대별 '오늘의 복약' 체크, 식약처 e약은요 API 실연동
- **갤러리** — 기록 사진·영상과 안심 갤러리(미아 대비) 통합
- **미래 준비** — 생애주기 4단계 × 5영역 목표, 단기/장기 구분
- **온보딩** — 첫 방문 둘러보기 가이드, 화면별 도움말

## 기술

**의존성 0개** — 프레임워크도 번들러도 없는 순수 HTML/CSS/JS(ES5). `npm install` 불필요.

| 구분 | 내용 |
|---|---|
| 라우팅 | 해시 기반 자체 구현 (`app.js`의 `ROUTES`) |
| 데이터 | localStorage(JSON 단일 문서) + IndexedDB(영상 클립만) — **백엔드 없음** |
| 폰트 | 나눔스퀘어 네오 5웨이트 + 나눔스퀘어라운드, 로컬 TTF 내장(오프라인 동작) |
| QR | 자체 구현 (`assets/js/qr.js` — GF(256)·리드-솔로몬 포함) |
| 외부 연동 | 식약처 e약은요 API(브라우저 직접 호출), 약학정보원 검색 링크 |
| 배포 | GitHub Pages (`main` 브랜치 루트) |

### 파일 구조

  views2.js             아이 프로필·등록·복용 관리·설명서 편집·요약 시트

## 로컬 실행

작업폴더 루트의 `serve.py`로 서빙합니다 (캐시 무효화 헤더 포함).

```bash
python serve.py 8767 --directory stellar-connect-mvp
```

`index.html` 더블클릭(`file://`)으로도 동작하지만, 그 경우 4개 트랙이 같은 origin을 공유해
**데모 데이터가 섞이므로** 시연·검증은 서버로 하세요.

## 개발 시 주의

- **배포 = 캐시버스팅**: 코드를 고쳤으면 `index.html`의 `?v=N`을 **15곳 모두** +1 해야
  사용자 화면에 반영됩니다. 현재 `v=96`
- **SEED_VERSION**: `seed.js`의 이 값을 올리면 모든 브라우저가 다음 접속에 재시드되어
  **사용자 입력 데이터가 지워집니다.** 데모 데이터를 고쳤을 때만 올리세요. 현재 `16`
- **인증키 커밋 금지**: 식약처 API 키는 브라우저 localStorage(`scon_drugApiKey`)에만 보관합니다
- 프로토타입이라 비밀번호는 평문 저장, 권한은 UI로만 강제됩니다 — 실서비스 전환 시 서버로 옮겨야 합니다
